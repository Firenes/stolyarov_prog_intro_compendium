Комбинаторика –– раздел математики, который отвечает на вопрос "сколько способов", "сколько вариантов" и т.д.

В комбинаторике рассматриваются конечные множества, с элементами которых что-то происходит: порядок, обрасывание, возврат, объединение, сортировка


Для учёта порядка

M_k = k * M_k-1

Прим. В дальнейшем обозначение M_k будет значить, что k должно быть в подстрочном стиле.

M – возможных перестановок
k – количество шаров

Если для k-1 шаров существует M_k-1 возомжных перестановок, то добавляя k-й шар, количество перестановок увеличим в k раз

k-1 – количество шаров после установки первого (как понимаю).

Само число k в данном случае называется факториалом – "k!"
Факториал натурального числа k считается "число перестановок из k элементов",
а то, что факториал равен произведению чисел от 1 до k – это следствие.


Для случая, когда объектов больше, чем мест

К примеру, есть 7 мест, но нужно расположить там 3 объекта.

7*6*5 = 210

В общем случае можно сделать так:

n!/(n-k)!

n – общее количество элементов множества
k – длина необходимого множества

Эту величину называют "количество размещений из n по k"

В русскоязычнй литературе обозначается: A_n+k (A из n по k)
В англоязычной литературе обозночается: (n)_k – убывающий факториал

Прим. так дальше будет обозначаться, где n – подстрочный символ, k – надстрочный


Для случая, когда нужно взять какое-то количество без учёта порядка

Случай, когда нужно взять k предметов из имеющихся n без учёта порядка – количество сочетаний из n по k.

Обозначается как C_n+k (Це из n по k) – C от слова combinations.

C_n+k = n!/k!(n-k)!

Отличие от общего случая A_n+k, где важен порядок и мы учитываем все перестановки, то в случае C_n+k, A_n+k больше в k! раз. Поэтому для получения нужно формулыA_n+k нужно поделить на k!. Т.к. нам нужно лишь необходимое число предметов.

Для общего понимания как подходить к решению таких задач вместо запоминания формул:

Общее число перестановок из n составляет n!,
но нужны только k членов факториала, 
так что лишние члены убираются, поделив на (n-k)!;
то, что получилось, есть количество комбинаций с учётом порядка.
Если порядок не волнует, то делим на k!,
т.к. каждую комбинацию посчитали k! раз.
