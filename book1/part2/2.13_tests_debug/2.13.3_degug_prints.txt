2.13.3 Отладочная печать

Отладочная печать – распечатанная информация, не относящаяся к задаче.

Отладочные сообщения должны быть легко узнаваемы и не должны сливаться с
остальной информацией, которую выдаёт программа.
Можно помечать начало сообщения с помощью слова "DEBUG".

Нельзя забывать о переводе строки после конца сообщения.
Дело в буферизации вывода. Сообщение, которое не закончилось переводом
строки, можно увидеть не сразу, а если программа завершилась аварийно, то
вообще не увидеть.

Операции вывода помещают информацию в буфер, из которого она отдаётся
операционной системе для выдачи в поток в определённых случаях:
- при заполнении буфера;
- при завершении программы;
- при вызове процедуры, принудительно очищающей буфер (во FreePascal –
  процедура flush; в частности flush(output) принудительно вытесняет буфер
  стандартного потока вывода).

При вывода на терминал, в отличии от вывода в файл, вытеснение производится
также при выдаче перевода строки и при затребовании программой операции
ввода – на случай, если перед этим было выдано приглашение к вводу.

Для отладочной печати правильно было бы воспользоваться директивой условной
компиляции.
Для условной компиляции используется символы, по которым можно определить
когда выполнять ту или иную операцию.
К примеру, для отладки можно использовать слово "DEBUG".
В начале программы указать директиву `{$DEFINE DEBUG}`, которая определяет
этот символ.
Использоваться в коде будет так:
```
{$IFDEF DEBUG}
writeln('DEBUG: x = ', x, ' y = ', y);
{$ENDIF}
```

Можно не указывать дерективу и с помощью флага во время компиляции задать
символ:
```
fpc -dDEBUG myprog.pas
```
